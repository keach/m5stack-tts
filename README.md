# m5stack-tts

M5Stack Basic（ESP32）向けのPlatformIOプロジェクトです。

電源投入時には3秒間のスプラッシュ画面を表示してから、SDカード、音声合成、Wi-Fiなどの初期化を開始します。スプラッシュ表示中にボタンBを長押しすると設定・診断モードへ入ります。

## 実装状況

### 実装済み

- PlatformIOによるM5Stack Basic向けのビルド、書き込み、シリアルモニター環境
- 電源投入時の3秒間のスプラッシュ画面
- Git管理対象外の設定ファイルを使ったWi-Fi接続とOpenWeather API設定
- 起動時のNTP時刻同期と日本標準時（JST）の表示
- OpenWeatherからの現在の天気、気温、湿度、気圧、直近1時間雨量の取得と画面表示
- OpenWeatherの5 day / 3 hour forecast APIから直近4件（約12時間分）の予報を取得・表示
- 起動時・10分ごとの自動更新と、ボタンAによる手動更新
- microSDカードの初期化と、天気情報の `/weather.csv` への記録
- 計測時刻を指定したAmbientへの気象データ送信
- AquesTalk ESP32 Small辞書版を使った日時・天気の読み上げ
- ボタンBによる表示中画面の読み上げ開始・停止と、ボタンCによる現在天気・予報画面の切替
- 30℃・35℃の高温アラート、夜間の消音、再通知制御、NVSへの状態保存
- 高温アラートの `/temperature_alerts.csv` への記録
- 現在の天気分類と1時間雨量を使った降雨アラート
- 時計表示と音量を保存できる設定モード
- アラーム音・音声のサウンドテストとサービス診断表示

## 初回セットアップ

Python 3.10以上を使用してください。

このプロジェクトでは、他のPlatformIOプロジェクトとフレームワークやツールチェーンが衝突しないよう、PlatformIO Coreの保存先をプロジェクト専用の `.pio-core` に分離しています。このディレクトリはGit管理対象外で、初回ビルド時に必要なパッケージがダウンロードされます。

```sh
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt
```

## Wi-Fi・天気設定

サンプルをコピーして、接続先のSSIDとパスワードを設定します。

```sh
cp include/secrets.example.h include/secrets.h
```

`include/secrets.h` にWi-FiのSSIDとパスワード、OpenWeather APIキー、天気を取得する地点の緯度・経度を設定してください。このファイルはGitの管理対象外です。

Ambientへ送信する場合は、サンプルをコピーし、作成したチャネルのチャネルIDとライトキーを設定します。このファイルもGitの管理対象外です。

```sh
cp include/ambient_secrets.example.h include/ambient_secrets.h
```

```cpp
constexpr unsigned int AMBIENT_CHANNEL_ID = 12345;
constexpr char AMBIENT_WRITE_KEY[] = "your-write-key";
```

Ambientには、`d1`から順に気温、湿度、気圧、直近1時間雨量、高温アラート閾値、現在の降雨状態、Wi-Fi RSSIを送信します。すべての送信でOpenWeather APIから正常に気象情報を取得した時刻を `created` として指定し、画面下部にも同じ時刻を `API Fetched` として `yyyy.mm.dd. hh:mm` 形式でAmbient送信結果とともに表示します。時刻が同期されていない場合はAmbient送信を行わず、SDカードへの記録は継続します。

Ambientへの送信に失敗したデータは、microSDの `/ambient_queue.ndjson` へ保存します。次回の天気更新時に新しいデータと合わせ、最大10件ずつAmbientの一括送信APIへ再送します。送信に成功したレコードだけをキューから除去します。応答が失われた場合などはAmbient側に同じ `created` のデータが重複する可能性があります。

起動時にWi-Fiへ接続した後、NTPサーバーから時刻を取得します。時刻は日本標準時（JST）で液晶とシリアルモニターに表示されます。液晶のデフォルト表示形式は `yyyy.mm.dd. ddd hh:mm` で、設定モードから秒表示へ切り替えられます。

現在の天気、気温、湿度、気圧、直近1時間の雨量と、直近4件の3時間予報は、起動時と10分ごとにOpenWeatherから取得します。ボタンAを押すと両方を手動更新できます。現在天気と予報は別々に取得するため、片方が失敗しても、もう片方の取得済みデータは引き続き利用します。ボタンAで更新した場合、次の自動更新はボタンを押してから10分後です。

## 予報画面

通常の天気画面でボタンCを押すと予報画面へ切り替わります。OpenWeatherの5 day / 3 hour forecast API（`/data/2.5/forecast`）から取得した直近4件について、予報時刻、天気分類、予想気温、降水確率、3時間予想雨量を表示します。画面切替だけではAPIへ再リクエストしません。

- ボタンA: 現在天気と予報を手動更新
- ボタンB: 表示中の画面を読み上げる。発話中は停止
- ボタンC: 現在天気画面と予報画面を切り替える

予報画面の下部には予報APIの取得成功時刻を `API Fetched` として表示します。1分間ボタン操作がない場合は、現在天気画面へ自動的に戻ります。予報の取得中、取得失敗、未取得も画面に表示し、更新に失敗した場合は直前に取得できた予報を保持します。

## SDカード

FAT32でフォーマットしたmicroSDカードを電源投入前に挿入してください。カードが認識されると、天気情報の取得成功時に `/weather.csv` へ次の形式で追記します。

```csv
datetime,weather,temp_c,humidity_pct,pressure_hpa,rain_1h_mm
2026-07-13 12:00:00,Clouds,28.4,72,1008,0.0
```

SDカードがない場合や初期化に失敗した場合も、時計と天気表示は継続します。

## AquesTalk ESP32 SDK

AquesTalk ESP32 Small辞書版2.4.4の評価版SDKは、再配布条件に従ってGit管理対象外の `vendor/aquestalk/aquestalk-esp32_s` に配置します。PlatformIOはこのローカルSDKを `lib_extra_dirs` から読み込みます。

漢字かな混じり文を音声合成するには、`sdcard` ディレクトリの内容をmicroSDカードのルートへコピーしてください。実機上の辞書パスは次のとおりです。

```text
/aq_dic/aqdic_m.bin
```

評価版には発音制限があります。継続的な開発や製品版としての利用には、AQUESTの使用ライセンスが必要です。

ライセンスキーを取得した場合は、Git管理対象外の `include/secrets.h` に設定します。空文字の場合は評価版として動作します。

```cpp
constexpr char AQUESTALK_LICENSE_KEY[] = "";
```

起動時に辞書と音声合成エンジンを初期化します。ボタンBを押すと表示中の現在天気または予報を内蔵スピーカーで読み上げ、発話中にもう一度ボタンBを押すと停止します。現在天気が未取得の場合は固定のテスト文、予報が未取得の場合は取得できない旨を読み上げます。

音声とアラーム音のデフォルト音量は50%です。設定モードから10%刻みで調整できます。

## 設定・診断モード

電源投入時のスプラッシュ画面でボタンBを1秒以上長押しすると、各サービスの初期化後に設定・診断モードへ入ります。短いボタンノイズによる誤操作を避けるため、メニュー操作も80ミリ秒以上の押下で確定します。

- ボタンA: 前の項目
- ボタンB: 選択・実行
- ボタンC: 次の項目

時計の分表示・秒表示、音量0〜100%（10%刻み）、アラーム音テスト、固定文の音声テスト、各サービスの診断結果を操作できます。`Save and exit` を選ぶと時計と音量の設定をNVSへ保存し、通常の天気画面へ戻ります。サウンドテストではアラートログや通知状態を変更しません。

診断画面では、microSD、AquesTalk辞書、音声サービス、Wi-Fi、NTP時刻、OpenWeather取得状態を確認できます。

## 高温アラート

現在気温が30℃または35℃以上になると、画面に警告を表示して `/temperature_alerts.csv` に記録します。6:00から24:00まではアラーム音と音声でも通知し、0:00から6:00までは画面表示とログ記録だけを行います。

各閾値は独立して管理されます。通知後、気温が閾値より1℃以上低くなり、かつ前回通知から3時間以上経過すると再通知可能になります。その後、再び閾値以上になると新しいアラートとして扱います。状態はESP32のNVSへ保存されるため、再起動後も引き継がれます。

```csv
datetime,temperature_c,threshold_c,audio_played
2026-07-15 14:00:00,30.2,30,true
2026-07-15 14:40:00,35.1,35,true
```

## 降雨アラート

OpenWeatherの現在天気が `Rain`、`Drizzle`、`Thunderstorm` のいずれかになったとき、非降雨から降雨への変化として1回だけ通知します。直近1時間雨量 `rain.1h` は、警告画面、音声、ログに使用します。

6:00から24:00まではアラーム音と「雨が降っています」という音声で通知し、0:00から6:00までは画面表示とログ記録だけを行います。同時に高温アラートが発生した場合は、高温アラートの音声を優先します。

雨でない観測が2回連続すると再通知可能になります。現在の更新間隔では約20分に相当します。降雨状態はNVSへ保存するため、雨が続いている間に再起動しても重複通知しません。

```csv
datetime,weather,rain_1h_mm,audio_played
2026-07-18 15:20:00,Rain,1.4,true
```

降雨ログは高温アラートとは別の `/rain_alerts.csv` へ記録します。高温と降雨が同時の場合、画面上部では高温警告を優先します。
